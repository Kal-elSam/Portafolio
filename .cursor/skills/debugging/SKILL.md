# Debugging Skill

Fuente de verdad:

- `docs/ai/harness.md`
- `docs/skills/debugging.md`

Usar cuando:

- hay bugs o errores
- comportamiento inesperado
- fallas de build o runtime

Procedimiento:

1. Definir esperado vs actual.
2. Reproducir y recolectar evidencia.
3. Crear regression test.
4. Corregir causa raíz.
5. Validar pipeline completo.
