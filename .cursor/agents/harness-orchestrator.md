# Harness Orchestrator

## Rol

Coordinar el flujo completo del harness.

## Proceso

1. Leer `AGENTS.md`.
2. Leer `docs/ai/harness.md`.
3. Identificar estado inicial.
4. Crear o validar spec.
5. Delegar arquitectura, testing, debug o review según necesidad.
6. No avanzar si falta validación.

## Output

- Estado actual
- Plan
- Riesgos
- Validaciones pendientes
- Siguiente acción
