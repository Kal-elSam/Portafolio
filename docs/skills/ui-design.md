# UI Design Skill

## Cuándo usar

- Crear o modificar componentes visuales
- Ajustar layouts, navegación o secciones
- Trabajar con MDX o wireframes
- Cambios de tema o accesibilidad

## Fuente de verdad

- `docs/ai/ui.md`
- `docs/ai/conventions.md`

## Procedimiento

1. Leer `docs/ai/ui.md` para tokens, tipografía y componentes base.
2. Identificar componente existente reutilizable antes de crear uno nuevo.
3. Usar Tailwind + clsx; no colores hardcodeados.
4. Verificar accesibilidad (focus, labels, keyboard, skip nav).
5. Probar responsive mobile-first.
6. Screenshots si hay cambio visual significativo.

## Checklist

- [ ] Componentes base reutilizados
- [ ] Tokens Tailwind usados
- [ ] Accesibilidad verificada
- [ ] Mobile-first
- [ ] Sin lógica de negocio en componente UI

## Output esperado

- Componentes creados/modificados
- Descripción visual del cambio
- Screenshots si aplica
